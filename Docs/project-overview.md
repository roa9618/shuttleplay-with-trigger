# 프로젝트 개요

셔틀플레이는 배드민턴 모임의 출석, 경기 매칭, 코트 운영, 경기 결과, 개인 운동 기록을 한곳에서 다루는 운영 도구입니다.

운영자는 모임과 운동 세션을 만들고 참가자를 관리합니다. 참가자는 QR 또는 공유 링크로 세션에 들어와 출석을 체크하고, 내 다음 경기와 현재 상태를 확인합니다. 시스템은 참가자의 성별, 급수, 나이대, 복식 MMR, 혼복 MMR, 경기 횟수, 휴식 상태, 파트너와 상대 중복 이력을 보고 경기 후보를 만듭니다.

핵심은 단순히 4명을 뽑아 경기를 만드는 것이 아닙니다. 특정 참가자가 오래 쉬지 않도록 하고, 같은 사람끼리만 반복해서 치지 않도록 하며, 신규 참가자가 너무 오래 기다리거나 바로 부담스러운 경기에 들어가지 않게 만드는 것이 중요합니다.

## 목표

- 운영자가 손으로 처리하던 출석, 지각, 코트 배정, 경기 기록 업무를 줄입니다.
- 참가자별 경기 기회가 지나치게 치우치지 않도록 합니다.
- 파트너와 상대 조합이 반복되는 문제를 줄입니다.
- 실력 차이가 큰 경기를 줄이고, 운영자가 납득할 수 있는 매칭 이유를 남깁니다.
- 경기 결과와 개인 기록을 누적해 이후 모임 운영에 활용합니다.
- 웹으로 빠르게 접속하되, PWA를 통해 앱처럼 사용할 수 있는 경험을 제공합니다.

## 사용자 역할

| 역할 | 설명 | 필요한 기능 |
| --- | --- | --- |
| 운영자 | 모임을 만들고 당일 운동을 진행하는 사용자 | 세션 생성, 참가자 관리, 출석 확인, 자동 매칭, 경기 시작/종료, 결과 입력 |
| 회원 참가자 | 로그인 후 개인 기록을 누적하는 사용자 | 세션 참여, 출석 체크, 내 경기 확인, MMR과 운동 기록 확인 |
| 비회원 참가자 | 회원가입 없이 당일 세션에 참여하는 사용자 | QR 접속, 기본 정보 입력, 출석 체크, 당일 경기 현황 확인 |
| 디스플레이 사용자 | 체육관 큰 화면에서 경기 현황을 보는 사용자 | 현재 경기, 다음 경기, 코트별 상태 확인 |

## 기본 흐름

```txt
모임 생성
→ 운동 세션 생성
→ 초대 링크 / QR 생성
→ 참가자 접속
→ 회원 또는 비회원 참여
→ 출석 체크
→ 참가자 상태 관리
→ 자동 매칭 생성
→ 경기 후보 큐 생성
→ 코트가 비면 경기 시작
→ 참가자에게 다음 경기 알림
→ 경기 종료
→ 승패 / 점수 입력
→ MMR 계산 및 기록 저장
→ 다음 경기 후보 재계산
→ 세션 종료
→ 요약 리포트 생성
```

## 1차 서비스 범위

- 모임 생성 및 관리
- 초대 링크와 QR 제공
- 회원/비회원 참가 등록
- 출석, 지각 예정, 불참 처리
- 참가자 상태 관리
- 자동 경기 매칭
- 경기 후보 큐 운영
- 코트별 경기 시작과 종료
- 경기 결과 입력과 수정
- 승패 및 점수 기반 MMR 계산
- 복식 MMR과 혼복 MMR 분리 관리
- 일자별 운동 기록
- 세션 종료 요약 리포트
- 운영자 대시보드
- 참가자 현황 화면
- 큰 화면 경기판
- 웹 내부 알림
- PWA 푸시 알림은 보조 알림으로 검토

## 제품 기준

- 필수 안내는 웹 내부 알림으로 처리합니다.
- PWA 푸시는 브라우저와 기기 제한이 있으므로 보조 수단으로 둡니다.
- 최초 화면 진입은 REST API로 현재 상태를 가져옵니다.
- 이후 상태 변경은 WebSocket 이벤트로 동기화합니다.
- WebSocket 연결이 끊기면 재연결 후 REST API로 최신 상태를 다시 맞춥니다.
- 운영자는 자동 매칭 결과를 그대로 쓰거나, 필요한 경우 후보를 수정할 수 있어야 합니다.
- 매칭 실패를 사용자에게 그대로 보여주기보다 가능한 최선의 후보를 만들고, 품질이 낮으면 운영자에게 알려야 합니다.
